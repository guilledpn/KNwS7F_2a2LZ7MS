# Programa de aprendizaje del CRM Patrimonial

Estado: Vigente  
Último LCD aprobado: LCD-20260713-01  
Aprobación: Pull Request #9

## Propósito

Integrar el aprendizaje profesional al trabajo real del proyecto. No es un curso separado: cada Issue, Pull Request, prueba, despliegue o decisión arquitectónica puede generar evidencia de competencia.

## Cinco niveles

1. Git y repositorio.
2. Colaboración y gestión.
3. Desarrollo y calidad.
4. Arquitectura y dominio.
5. Operación.

## Estados de competencia

- **Pendiente:** aún no abordado.
- **Introducido:** reconoce el concepto y su propósito.
- **Guiado:** puede aplicarlo siguiendo instrucciones.
- **Operativo:** puede aplicarlo en situaciones habituales.
- **Autónomo:** decide y ejecuta sin ayuda relevante.
- **Sólido:** puede explicarlo, detectar errores y escoger cuándo utilizarlo.

## Regla de evidencia

Una competencia cambia de estado por evidencia práctica, no sólo porque haya sido explicada.

Evidencias posibles:

- ejecutar correctamente una acción;
- tomar una decisión adecuada;
- explicar un concepto con palabras propias;
- detectar un error o riesgo;
- completar un flujo real;
- resolver un problema con ayuda decreciente.

## Ritmo de revisión

- revisión breve al cerrar una tarea relevante;
- revisión por LCD o Release;
- revisión general periódica de fortalezas, debilidades y pendientes.

## Archivos

- `skills-matrix.md`: estado actual de competencias.
- `learning-log.md`: registro cronológico de prácticas y evidencias.
